## Route / nav findings (from portal-route-auditor)

**Verdict: FAIL — 6 critical, 5 high, 6 medium, 5 low.**

### Critical
1. **Manifest-vs-code path drift on 4 ops routes** — `route-manifest.json:15-18` declares `/ops/stock/{receipts,waste-adjustments,physical-count,production-actual}` as `live`, but the actual pages live at `/stock/...` (Next.js `(ops)` group, parens stripped). `src/app/(ops)/stock/receipts/page.tsx` and siblings; `src/lib/nav/manifest.ts:104,110,117,124` confirms `/stock/...`. **Manifest is wrong on its 4 most-used operator routes.**
2. **`/admin/jobs` rendered as `<QuarantinedPage>`** but `route-manifest.json:32` says `live` and `src/lib/nav/manifest.ts:237-243` exposes it in SideNav. Direct violation of invariant #3 (quarantined surfaces in primary nav). Source: `src/app/(admin)/admin/jobs/page.tsx:1-5`.
3. **`/admin/users` rendered as `<QuarantinedPage>`** — same pattern. `src/app/(admin)/admin/users/page.tsx:1-5` + `src/lib/nav/manifest.ts:230-236`. Manifest line 33.

### High
- **`/inbox` is in code + nav but absent from manifest** — `src/app/(inbox)/inbox/page.tsx:23`, `src/lib/nav/manifest.ts:80-86`. Already flagged in the manifest's own `_todo_after_bootstrap`.
- **`/admin/products/[item_id]` orphan** — `(admin)/admin/items/[item_id]/page.tsx:13-20` redirects to it; manifest does not enumerate the `products` namespace at all.
- **Forbidden_string re-entry: `X-Fake-Session` / `X-Test-Session`** in a comment at `src/lib/api-proxy.ts:14`. `quarantine.json` forbids both with no exemption.
- **Forbidden_string re-entry: `FakeSession`** identifier used extensively in `src/lib/auth/fake-auth.ts:26,30,36,70,73,85,94,105` and `src/lib/auth/session-provider.tsx:31,36,41,52,59,85`. File self-documents as deliberately retained but no `pending-cleanup` quarantine entry exists.
- **Forbidden_string re-entry: `fakeAuth`** in `src/lib/auth/fake-auth.ts:5,90` (`STORAGE_KEY="gt.fakeauth.v1"`, `isFakeAuthEnabled`).

### Medium
- `/admin/items/[item_id]` legacy redirect not in manifest.
- `/planning/forecast/[version_id]`, `/planning/forecast/new`, `/planning/runs/[run_id]` sub-pages exist in code, manifest only declares parents.
- `/profile`, `/exceptions`, `/dashboard` role-gate semantics differ between manifest and per-group `layout.tsx` (`(shared)/layout.tsx:9` admits any auth via `viewer:read`; `(planner)/layout.tsx:8` admits viewer to `/exceptions`).

### Low
- Nav drift: planned move `/admin/sku-aliases` → `/admin/integrations/sku-aliases` carried as comment only (`src/lib/nav/manifest.ts:252-254`).
- `/stock/submissions`, `/admin/signals` — planned URLs in nav comments (lines 131-135, 261-262), no rows.
- Middleware (`src/middleware.ts:43-48`) is auth-only — no path-specific role gates; relies on layout RoleGates.
- Middleware bypass when `NEXT_PUBLIC_ENABLE_DEV_SHIM_AUTH=true` (`src/middleware.ts:35-37`); production-cutover risk.
- TopBar "FAKE SESSION" pill is unconditional render (`src/components/layout/TopBar.tsx:74-118`).

## Suggested next tranche focus

Two candidate slugs for `/portal-tranche-plan` to choose between:

**Candidate A — `bootstrap-truthfulness` (recommended first):** Tranche 001 + 002 + 003 + 008 as one bounded set. Rationale: every other gap is masked or amplified by the manifest lying, the integrations dashboard fabricating, and the QuarantinedPage admin shells claiming live status. Until the OS's *own* truth files agree with `src/`, no scorecard or regression-sentinel can do its job. This is the single highest-leverage tranche and is purely **read-with-a-pen** — no new product surface, only files under `docs/portal-os/`, `src/app/(admin)/admin/{integrations,jobs,users}/page.tsx`, `src/lib/nav/manifest.ts`, `src/lib/auth/fake-auth.ts`, `src/lib/auth/session-provider.tsx`, `src/lib/api-proxy.ts`.

**Candidate B — `stock-readback-and-inbox` (recommended second):** Tranche 005 + 006 + 007. Rationale: all four operator surfaces currently emit data into a void — the operator never sees their own work again, and planners must guess submission IDs to approve anything. This is the largest single user-experience improvement available. Build the `/stock/submissions` list, the unified inbox listing, deep-link approval banners, and wire the Physical Count Cancel proxy.

Tranches 004, 009, 010 should follow once A + B are landed.
